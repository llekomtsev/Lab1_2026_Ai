# Лабораторная работа 1: Классификация пород собак (Stanford Dogs) с использованием SqueezeNet

## Цель работы
Научиться реализовывать алгоритм глубокого обучения, провести эксперименты по дообучению предобученной модели и обучению модели с нуля, сравнить их качество.

## Датасет
Stanford Dogs Dataset — 120 пород собак, всего около 20 580 изображений.  
Разделение:  
- Обучающая выборка: 70%  
- Валидационная выборка: 15%  
- Тестовая выборка: 15%  

## Архитектура и методы
- **Архитектура:** SqueezeNet (предобученная на ImageNet и инициализированная случайно для обучения с нуля).  
- **Аугментации:** Gaussian Blur (случайно с вероятностью 0.5), ресайз до 224×224, нормализация (среднее и стандартное отклонение ImageNet).  
- **Оптимизаторы:** Adam (с `amsgrad=False`) и AmsGrad (Adam с `amsgrad=True`) для дообучения; Adam для обучения с нуля.  
- **Функция потерь:** Cross‑Entropy Loss.

## Эксперименты
1. **Дообучение предобученной модели + Adam**  
2. **Дообучение предобученной модели + AmsGrad**  
3. **Обучение модели с нуля + Adam**

Все модели обучались в течение 10 эпох с размером батча 64. Обучение проводилось в Google Colab с использованием GPU.

## Результаты

| Эксперимент                     | Precision | Recall | F1-score |
|---------------------------------|-----------|--------|----------|
| Pretrained + Adam               | 0.5843    | 0.5563 | 0.5526   |
| Pretrained + AmsGrad            | 0.5729    | 0.5466 | 0.5450   |
| From Scratch + Adam             | 0.0001    | 0.0117 | 0.0003   |

## Выводы
- **Transfer learning** даёт огромный выигрыш в качестве: предобученные модели достигают F1 ≈ 0.55, тогда как обучение с нуля практически не обучается (F1 ≈ 0.0003).  
- Разница между Adam и AmsGrad в условиях данной задачи незначительна; AmsGrad показал результат на 0.0076 ниже по F1.  
- Обучение с нуля на таком небольшом датасете и при ограниченных вычислительных ресурсах (10 эпох) не позволяет модели изучить сложную структуру 120 классов.  
- Таким образом, для задач классификации с ограниченным объёмом данных использование предобученных моделей является критически важным.


## Дополнительное исследование: влияние усиленных аугментаций

Поскольку в основном варианте использовалась только аугментация Gaussian Blur, был проведён дополнительный эксперимент, чтобы оценить, как более сильные преобразования влияют на качество моделей. В этом эксперименте:

- Для обучения использованы аугментации: `RandomResizedCrop`, `RandomHorizontalFlip`, `RandomRotation(10)`, `ColorJitter`, а Gaussian Blur сохранён.
- Learning rate для обучения с нуля уменьшен с 1e-3 до 1e-4 (для предотвращения расходимости).
- Все остальные параметры (размер батча, число эпох, нормализация) остались без изменений.

**Результаты дополнительного эксперимента:**

| Эксперимент                     | Precision | Recall | F1-score |
|---------------------------------|-----------|--------|----------|
| Pretrained + Adam               | 0.5321    | 0.4896 | 0.4766   |
| Pretrained + AmsGrad            | 0.5511    | 0.4961 | 0.4941   |
| From Scratch + Adam             | 0.0072    | 0.0188 | 0.0075   |

**Наблюдения:**
- Качество предобученных моделей снизилось по сравнению с базовыми настройками (F1 ~0.48–0.49 против ~0.55). Вероятно, это связано с тем, что сильные аугментации сильно меняют распределение данных, и для адаптации потребовалось бы больше эпох.
- AmsGrad в новых условиях показал результат немного выше обычного Adam (0.4941 против 0.4766).
- Обучение с нуля улучшилось с F1=0.0003 до 0.0075, но всё ещё остаётся крайне низким.

**Вывод:** даже при более сложных аугментациях предобученные модели значительно превосходят модель, обученную с нуля. Это подтверждает, что использование предобученных весов является ключевым фактором успеха в задачах классификации с ограниченным объёмом данных.
